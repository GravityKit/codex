◆ check_approved_identifier()

check_approved_identifier (   $secret_id)
private

Makes double-y sure the TrustedLogin Server approves this support-agent login.

This function sends server variables to the TrustedLogin server to help prevent a number of attack vectors. It is only ever triggered as part of the auto-login sequence. The session data synced will only ever be from authorized support teams, or potential attackers.

Parameters
string$secret_idThe secret ID for the site.
Returns
true|WP_Error True: the TrustedLogin service was reached and the login remains valid. WP_Error: The service wasn't reachable or the service responded that the secret ID wasn't valid.

This array contains information from the Vendor's support agent as a means of protecting against potential breaches.

No site user/visitor/admin data is sent back to TrustedLogin server.

◆ in_local_development()

in_local_development ( )
private

Is this site in local development mode?

() If available, used to fetch site's development environment

See also
https://developer.wordpress.org/reference/functions/wp_get_environment_type/

To bypass lockdown checks, set a WordPress environment to local or development. Alternately, you may add a constant to the site's wp-config.php file formatted as TRUSTEDLOGIN_TESTING_{EXAMPLE} where {EXAMPLE} is replaced with the project's upper-cased namespace.

Returns
bool True: site is in local or development environment. False: site is live.
