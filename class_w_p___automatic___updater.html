is_vcs_checkout (   $context)

Check for version control checkouts.

Checks for Subversion, Git, Mercurial, and Bazaar. It recursively looks up the filesystem to the top of the drive, erring on the side of detecting a VCS checkout somewhere.

ABSPATH is always checked in addition to whatever $context is (which may be the wp-content directory, for example). The underlying assumption is that if you are using version control anywhere, then you should be making decisions for how things get updated.

Since
3.7.0 public

send_core_update_notification_email (   $item)
protected

Notifies an administrator of a core update.

Since
3.7.0 protected
Parameters
object$itemThe update offer.

Filter whether to notify the site administrator of a new core update.

By default, administrators are notified when the update offer received from WordPress.org sets a particular flag. This allows some discretion in if and when to notify.

This filter is only evaluated once per release. If the same email address was already notified of the same new version, WordPress won't repeatedly email the administrator.

This filter is also used on about.php to check if a plugin has disabled these notifications.

Since
3.7.0
Parameters
bool$notifyWhether the site administrator is notified.
object$itemThe update offer.

Filter whether to automatically update core, a plugin, a theme, or a language.

The dynamic portion of the hook name, $type, refers to the type of update being checked. Can be 'core', 'theme', 'plugin', or 'translation'.

Generally speaking, plugins, themes, and major core versions are not updated by default, while translations and minor and development versions for core are updated by default.
